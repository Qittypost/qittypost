Url

Creation

To create URL buttons, go to Buttons -> URL.

Syntax

The button URL has a simple format:

btn1 - url1 | btn2 - url2
btn3 - url3
  • Use the symbol - to separate the button name from the link.
  • Use the symbol | to place multiple buttons in the same line.
  • Use a newline to place buttons on separate lines.

Escaping

WARNING: All of the following examples use text as the value, but in reality, it can only be a link; otherwise, an error will occur!!!
It is also forbidden to use the folowing characters: ", |, -, or any whitespace characters in the name or value.
But you can use these symbols by enclosing the name or the value in the double quotes.

// Wrong solution

Using -| denied symbols in the name - ... | ...
...

// Correct solution

"Using -| denied symbols in the name" - ... | ...
...

// Correct solution

"Using -| denied symbols in the name" - "..." | ...
...

Note: You still cannot use the " character in the text directly. To include it, you need to escape it using the following format: \".
The use of the \ character is a bit specific in both name and value. You can't use it at the end of a line.

// Wrong solution

"Backslash symbol in the end of the line\" - "..." | ...
...

Instead you have to double it \\.

// Correct solution

"Two backslash symbols in the end of the line\\" - "..." | ...
...

In this case \\ will be replaced by \. So the result will be button with the name Two backslash symbols in the end of the line\, with value It is correct\.
Also, you cannot use an odd number of backslashes at the end of a line.
Note: You can use any number of backslashes in the start of the line.
Note: In any case, the characters \\ will be replaced by \, so \\ becomes \, and \\\\ becomes \\, and so on.
You can also use a single backslash in text.

// Correct solution

"single \ backslash" - "..." | ...
...

But as we know from the previous case, double backslashes are interpreted as a single backslash. Therefore, if you want to use two or more backslashes in a row, you should double them.

// Correct solution

"two backslashes \\\\ in a row" - "..." | ...
...

Note: The last backslash can be single. For example, "two \\\ backslashes" will be interpreted as two \\ backslashes.

Using backslash with " is also specific.
As you know, using " directly in the name or value is not allowed; instead, you simply need to escape this symbol using the \" construction. From this, certain peculiarities of using \ before the " symbol arise.
You cannot use an even number of backslashes before ".

// Wrong solution

"Even number of \\\\" backslashes" - "..." | ...
...

// Correct solution

"Odd number of \\\\\" backslashes - "..." | ...
...

Using backslashes outside of double quotes works the same as inside them, except that you can now use any number of backslashes at the end.

// Correct solution

\\\Using_backslash_\in_the_end_is_correct\\\ - ... | ...
...

Note: Although the use of " is completely prohibited outside of double quotes, meaning there is nothing to escape, but even here the rules for replacing backslashes remain the same as before. Specifically, in any case, the characters \\ will be replaced by \, so \\\ becomes \, and \\\\ becomes \\, and so on.
